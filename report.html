New Flag Columns

欄位DescriptionWhy Add It
income_missingWas income originally missing? (0/1)Missing group has lower default rate — missingness is itself a signal
income_zeroWas income originally 0? (0/1)0 may indicate unemployment or data error — preserve this info
has_dependentsAre there dependents declared? (0/1)Having dependents signals financial burden
⚠ Flag columns must be added BEFORE filling missing values to capture the original missing state. Adding flags after imputation would make all flags = 0, losing meaning.

CategoryCountDescription
✅ Approved Good Customers23,122 人Correctly approved, generating profit

✅ 核心建議：在損益比 1:10 的假設下，建議將核貸門檻設在違約機率 0.50。當申請人預測違約機率 ≥ 50% 時拒絕放款，可讓公司整體損益最大化。

門檻不是越嚴越好——門檻過高會拒絕太多好客戶，損失業務收入。實際業務應以真實損益數字替換Assumed Value，Optimal Threshold會隨之調整。
